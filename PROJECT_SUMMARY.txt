## 📊 API Endpoints

### Agent Demo Endpoints (with Live Data)
- `POST /demonstrate-news-agent` - Fetch live tech news (6+ articles)
- `POST /demonstrate-research-agent` - Get arXiv papers (6+ papers)
- `POST /demonstrate-task-agent` - Create task with dependencies
- `POST /demonstrate-scheduler-agent` - Schedule events
- `POST /demonstrate-knowledge-agent` - Gather context analysis
- `POST /demonstrate-critic-agent` - Workflow optimization analysis

### Agent Control Endpoints
- `GET /health` - System health check (fast, <100ms)
- `GET /api/agents/status` - All agent statuses (no database calls)
- `GET /api/agents/status/{agent_name}` - Individual agent info

### Data Management
- `GET /api/tasks` - List all tasks (with mock fallback)
- `POST /api/tasks` - Create new task
- `PUT /api/tasks/{task_id}` - Update task
- `DELETE /api/tasks/{task_id}` - Delete task
- `GET /api/events/upcoming/{days}` - Get upcoming events
- `POST /api/events` - Create event
- `GET /api/mock/tasks` - Mock task data
- `GET /api/mock/events` - Mock event data

### Workflow Orchestration
- `POST /workflows` - Create workflow
- `GET /workflows/{workflow_id}` - Get workflow status
- `GET /workflows/{workflow_id}/audit` - Critic audit report

### Safety & Consensus
- `POST /actions/vibe-check` - Cross-agent safety check
- `POST /debate/initiate` - Multi-agent debate
- `GET /debate/{debate_id}` - Debate results
- `GET /vibe-check/{check_id}` - Safety audit report
